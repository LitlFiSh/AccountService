# 后端接口

## 1. 登录

   ```json
url:"/user/login"
method:"POST"
params:"username", "password"
//username: 用户id，password: 用户密码

//返回信息说明
//1. 登录失败：在返回体中返回失败消息
//2. 登陆成功：返回成功消息，并且在返回体头部中设置了字段为 Authorization 的 token 信息，之后用户每次请求操作都需要携带该参数

//返回值
{
    "deptName": "" //用户所在部门名称
    "loginName": "" //用户名（真实姓名）
    "role": 4, //用户ID，（普通用户）
    "menu":[
        {
            "name"
            "path"
            "children":[
            	{
            		"name"
            		"path"
            		"children": []
        		}
            ]
        },
    ]
}
   ```

==注：token的有效时间为小时，如果后台验证token距离过期时间小于10分钟，则重新签发新token，新token放在该请求的返回体头部，签发新token后，原来的token会失效。==

## 2. 用户操作

#### 2.1 查找用户信息：

```json
url: "/user/info"
method: "GET"

{
    "id":"" //用户id
    "username":"" //用户真实姓名
    "department":"" //用户所属部门名称
    "role":"" //用户拥有角色名称
    "inCharge":"" //是否采购负责人(true/false)
}
```

#### 2.2 退出登录：

```json
url: "/user/logout"
method: "GET"
params: "uid"
//uid: 用户ID

//说明：用户点击退出后需要发送该请求，将后台缓存中的 token 销毁
```

#### 2.3 修改用户密码：

```json
url: "/user/pwd"
method: "PUT"
data:
	"uid" //用户id
	"oldPassword" //旧密码
	"newPassword" //新密码

//说明：成功修改密码后应自动退出登录
```

#### 2.4 模糊搜索用户名

```json
url: "/user/search"
method: "GET"
params: "name"
//name: 用户名
//例：name=李，搜索所有名字中带有'李'的用户

//返回值
[
    {
        "username", //用户姓名
        "department", //用户所属部门名称
    }
]
```

## 4. 申请单操作

#### 4.1 新增申请

```json
url: "/order"
method: "POST"
data:
//orderApply
{
    applyDepartment //申请部门，通过选择从服务器获取的所有部门信息进行赋值，值为对应部门的名称
    applyUser //申请人
    fundCode //采购经费代码
    total //采购总金额
    status //申请单状态：-1->已删除，0->待修改，1->提交，2->部门领导通过，3->主管领导通过...
    orderLists //申请设备列表
    uid //当前登录用户id
}
//orderList
{
    name //物资名称
    type //品牌型号
    configuration //配置或技术参数
    unit //单位
    quantity //数量
    budgetUnitPrice //预算单价
    budgetTotalPrice //预算总价
    reason //申请原因
    newUser //新使用人
}
```

#### 4.2 更新申请

```json
url: "/order"
method: "PUT"
data:
//orderApply
{
    applyDepartment //申请部门，通过选择从服务器获取的所有部门信息进行赋值，值为对应部门的名称
    applyUser //申请人
    fundCode //采购经费代码
    total //采购总金额
    status //申请单状态：-1->已删除，0->待修改(保存)，1->提交，2->部门领导通过，3->主管领导通过...
    orderLists //申请设备列表
    uid //当前登录用户id
}
```

#### 4.3 撤销提交

```json

//例：http://localhost:8080/order/orders?oid=2021&startDate=2021-02-01&page=1

    "content": [

			"id", //申请单id

			"applyUser",  //申请人姓名

			"status", //申请单状态
			"orderLists": [
				{
					"id", //设备列表id
					"name", //物资名称

					"configuration", //配置或技术参数
					"unit", //单位
 					"quantity", //数量
					"budgetUnitPrice", //预算单价
					"budgetTotalPrice", //预算总价
					"reason", //申购原因
					"newUser" //新设备使用人
				}

			"uid", //申请单对应用户id
			"withdrawalReason", //打回原因，部门领导或主管院领导打回该申请单时须填写的内容
			"createTime", //申请单创建时间
			"updateTime" //申请单最后更新时间
		},

	"totalElements", //总数量
	"size", //每页数量
	"number", //当前页数
}
```

### 返回数据说明

#### 申请单(orderApply)

```json
{
    id, //申请单id，由服务器在新建申请单时生成
    applyDepartment*, //申请部门，通过选择从服务器获取的所有部门信息进行赋值，值为对应部门的名称
    applyUser*, //申请人
    fundCode*, //采购经费代码
    applyDate, //申请日期，在用户提交后由服务器生成
    total*, //采购总金额
    status*, //申请单状态：-1->已删除，0->待修改，1->提交，2->部门领导通过，3->主管领导通过...
    orderLists*, //申请设备列表
    uid*, //申请单对应用户id，一般为当前登录用户id
    withdrawalReason, //打回原因，部门领导或主管院领导打回该申请单时须填写的内容
    createTime, //申请单创建日期，由数据库自动维护
    updateTime, //申请单最后更新日期，由数据库自动维护
}
注：带 * 的字段需要在提交申请单时携带。
申请单状态(status)：用户点击提交时，将 status 设置为 1 并发送 POST 请求；
			用户点击保存时，将 status 设置为 0 并发送请求。
```

#### 设备列表(OrderList)


```json
{
    id, //设备列表id，通过申请单id生成，由服务器在新建申请单时生成
    name*, //物资名称
    type*, //品牌型号
    configuration*, //配置或技术参数
    unit*, //单位
    quantity*, //数量
    budgetUnitPrice*, //预算单价
    budgetTotalPrice*, //预算总价
    reason*, //申请原因
    newUser*, //新使用人
}
注：带 * 的字段需要在提交申请单时携带。
注意，在新建申请单操作时，用户每添加一条设备列表时，对应的 status 都应设置为 1。
```

#### 用户(ResultUser)

### 代码示例

```js
//下载 Excel 文件
this.$axios
  .get("/order/file", {
    params: {
      id: this.orderApply.id,
    },
    responseType: "blob",   //文件下载的 url 需要带上这个参数
    headers: {
      Authorization: this.$store.state.token,
    },
  }).then((res) => {
     console.log(res);
     const { data, headers } = res;
     const fileName = headers["content-disposition"].replace(
       /\w+;filename=(.*)/,
          "$1"
        );   //根据返回头的content-disposition字段中的参数决定文件名
  	  //content-type 决定文件类型
        const blob = new Blob([data], { type: headers["content-type"] });
  	  //下载文件方式：在 html 中插入一个不可见的 a 标签，将返回的文件连接到 a 标签上实现下载
        let dom = document.createElement("a");
        let url = window.URL.createObjectURL(blob);
        dom.href = url;
        dom.download = decodeURI(fileName);
        dom.style.display = "none";
        document.body.appendChild(dom);
        dom.click();
        dom.parentNode.removeChild(dom);
        window.URL.revokeObjectURL(url);
      })
      .catch((err) => {
        console.log(err);
      });
```
